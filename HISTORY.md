# История изменений

## 0.1.0-alpha.3

- Реализованы восемь наблюдаемых исходов выполнения, единые deadlines/retries и
  детерминированное разделение пустого результата, подтвержденного нуля,
  частичного ответа, ошибки запроса и недоступности зависимостей.
- Добавлены required/optional closure плана, Evidence 1.1 и проверка полноты
  коллекции отдельно от валидности фактов; чтение замороженного Evidence 1.0
  осталось совместимым без переписывания исходных артефактов.
- Переносимые списочные навыки переведены на проверяемую keyset-пагинацию с
  одноразовыми continuation handles; импорт отвергает неполный sort/cursor,
  nullable coordinates и запрос без доказуемого lexicographic predicate.
- Добавлены универсальный resolver складов, список реализаций и первая
  композиция «склады по условию -> остатки», без ветвления application-кода по
  объектам УТ или идентификаторам тестовых вопросов.
- Реализованы database marker, двухфазная очистка с preview/confirm,
  диагностические контракты и миграция SQLite `0003_slice2`.
- Независимый acceptance-набор второго среза проверяет outcomes, пустые/null
  ответы, 43 объекта из 86 строк регистра без дублей, пагинацию, продолжения,
  перенос пакетов, hot reload и отказоустойчивость.
- Приняты 448 автоматических тестов, 64 black-box проверки второго среза и
  сохранены 14/14 сценариев первого E2E; Ruff, строгий Mypy, lock, wheel и sdist
  проходят. Два wall-clock TTL-теста и live MCP smoke остаются внешними воротами.

## 0.1.0-alpha.2

- Реализован минимальный сквозной продуктовый срез: русскоязычный web-чат,
  FastAPI, CLI, SQLite/Alembic, DeepSeek planner и read-only MCP adapter.
- Добавлен переносимый starter catalog из восьми навыков УТ: встроенная
  справка, четыре варианта поиска номенклатуры, остатки, заголовок/статус и
  строки заказа клиента.
- Импорт, замена, удаление и экспорт bare/embedded-closure JSON-навыков
  выполняются атомарно и начинают действовать без перезапуска приложения.
- Контекст хранит exact `_objectRef` вместе с происхождением факта и digest
  producer-навыка; подмена UUID, физического типа или semantic type
  отклоняется до MCP.
- Добавлены индекс встроенной справки УТ, детерминированный renderer,
  диагностический ZIP, SSE-прогресс и защита от чужого echo планировщика.
- Исправлена нормализация MCP Streamable HTTP ошибок, включая вложенные
  `ExceptionGroup`, без преобразования транспортного сбоя во внутреннюю ошибку.
- Приняты 356 автоматических тестов и отдельный black-box прогон из 14 E2E
  сценариев через два чистых экземпляра приложения и fixture DeepSeek/MCP.

## 0.1.0-alpha.1

- Реализовано первое production-ядро: закрытые Pydantic-модели plan, evidence,
  skill и skill package, bounded JSON loader и Draft 2020-12 validation.
- Добавлен детерминированный lexer/shallow parser языка запросов 1С для одного
  SELECT и связанных пакетов с временными таблицами без операций записи.
- Типизированы допустимые инвариантные литералы; бизнес-значения разрешены
  только через параметры навыка.
- Реализованы проверки покрытия фактов плана, точного output/provides,
  dependency lock, digest conflicts и расхождений документальных источников.
- Добавлен ADR-0003, независимые fixtures/probes и production contract tests.
- Зафиксировано 310 успешных тестов; один live MCP test остается
  `blocked_external` до запуска локального MCP-сервера.
- Сборка wheel включает все четыре канонические JSON Schema; Ruff, строгий
  Mypy, lock check и сборка пакета проходят успешно.
