# История изменений

## 0.1.0-alpha.1

- Реализовано первое production-ядро: закрытые Pydantic-модели plan, evidence,
  skill и skill package, bounded JSON loader и Draft 2020-12 validation.
- Добавлен детерминированный lexer/shallow parser языка запросов 1С для одного
  SELECT и связанных пакетов с временными таблицами без операций записи.
- Типизированы допустимые инвариантные литералы; бизнес-значения разрешены
  только через параметры навыка.
- Реализованы проверки покрытия фактов плана, точного output/provides,
  dependency lock, digest conflicts и расхождений документальных источников.
- Добавлен ADR-0003, независимые fixtures/probes и production contract tests.
- Зафиксировано 310 успешных тестов; один live MCP test остается
  `blocked_external` до запуска локального MCP-сервера.
- Сборка wheel включает все четыре канонические JSON Schema; Ruff, строгий
  Mypy, lock check и сборка пакета проходят успешно.

## 0.1.0-planning.7

- Принят независимый тестовый контракт до начала production-разработки.
- Добавлены 67 автономных проверок требований, JSON Schema и архитектурных
  инвариантов, а также валидные и отрицательные contract fixtures.
- Зафиксированы release gates, oracle model и checklist живой среды.
- Устранены противоречия baseline: обязательны все 87 возможностей, а
  performance-набор однозначно связан с десятью сценариями корпуса.
- Подтвержден DeepSeek JSON-mode smoke; live MCP-проверки оставлены со статусом
  `blocked_external` до запуска сервера.
