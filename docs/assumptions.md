# Исходные допущения и вопросы

## Принятые допущения

1. Целевая конфигурация MVP — типовая демонстрационная
   `1С:Управление торговлей (базовая), редакция 11`, релиз `11.5.27.56`,
   режим совместимости `8.3.27`, представленная файловой выгрузкой
   `/Users/repnikov/Documents/УТ_Демо/config`.
2. Для живых проверок используется известный MCP-сервер 1С, локально доступный
   по адресу `http://127.0.0.1:6003`.
3. Основная LLM — DeepSeek через OpenAI-compatible API; ключ хранится только в
   локальном окружении.
4. Пользовательский интерфейс MVP — web-клиент для диалога, диагностики и
   просмотра подключенного набора навыков.
5. Целевая серверная среда поставки — Windows; разработка и ранние проверки
   могут выполняться на macOS.
6. Работа с данными 1С в MVP только на чтение.
7. Безопасность ограничивается базовой эксплуатационной гигиеной прототипа;
   отдельная корпоративная модель угроз не входит в MVP.
8. Код и архитектура версии 5 не копируются. Разрешено использовать ее
   документацию, тестовые вопросы и наблюдения о неудачных решениях.
9. Пакет навыков предназначен для экземпляров одной конфигурации и совместимой
   структуры данных; межконфигурационная переносимость не является целью MVP.
10. Основной источник документации MVP — встроенная справка целевой выгрузки
    УТ. Дополнительные источники должны быть явно обозначены и не могут
    незаметно подменять документацию целевого релиза.
11. Контрольные результаты получают на локальной демонстрационной базе УТ и
    сохраняют вместе с идентификатором состояния тестовых данных. Недоступность
    живого MCP блокирует интеграционные испытания, но не анализ, архитектуру и
    автономную разработку.
12. Параметры DeepSeek задаются окружением. Их корректность подтверждается
    отдельным smoke-тестом, который не выводит и не сохраняет API key.

## Принятые обратимые решения MVP

1. Пользовательская авторизация не входит в прототип; используется локальный
   тестовый доступ.
2. Один процесс приложения работает с одной информационной базой.
3. Web-интерфейс позволяет просматривать, импортировать, экспортировать,
   заменять и удалять навыки. Визуальный редактор JSON в MVP не требуется.
4. Лимит списка по умолчанию — 20 строк. Ответ сообщает об ограничении и
   позволяет запросить продолжение или другой лимит.
5. Сессии и диагностические трассы сохраняются до ручной очистки.
6. Идентификаторы capability и навыков доступны в деталях и диагностике, но не
   загромождают обычный пользовательский ответ.
7. Сырые запросы и ответы MCP доступны только в диагностике.
8. Первый релиз использует только встроенную справку целевой УТ.
9. Переносимость навыка проверяется через web-интерфейс и файловую/CLI
   операцию.
10. Характеристики, серии и назначения могут быть типизированными параметрами
    навыков, но не требуют отдельного классификатора вопросов или ветвей кода.
