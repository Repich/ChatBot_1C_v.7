# Каталог продуктовых возможностей MVP

## 1. Назначение каталога

Этот документ описывает бизнес-возможности, которые должен предоставлять
готовый набор навыков ChatBot 1C v7.

Обозначения `CAP-*` являются **product capability IDs**: стабильными ссылками
из требований и тестовых сценариев на ожидаемую возможность продукта. Они не
задают имена файлов, внутренние идентификаторы навыков, способ исполнения,
структуру JSON или устройство приложения. Эти решения принимает архитектор.

Все 87 product capability IDs этого каталога образуют обязательный baseline
MVP. Возможность нельзя исключить решением во время приемки. Изменение состава
baseline требует отдельного принятого изменения требований и версии корпуса.

## 2. Требования к атомарности

1. Одна возможность выполняет одну проверяемую бизнес-операцию.
2. Товар, склад, клиент, организация, период, статус и лимит являются
   изменяемыми входными значениями.
3. Поиск объекта не объединяется с произвольным набором несвязанных показателей.
4. Получение факта отделяется от производного бизнес-расчета.
5. Результат имеет однозначный бизнес-смысл и известные единицы измерения.
6. Конкретные бизнес-значения не входят в identity возможности, правило ее
   выбора, шаблон запроса или фиксированный runtime binding.
7. Переносимые examples и fixtures могут использовать только явно
   синтетические значения и не влияют на выбор или исполнение возможности.
8. Секреты и локальные пути запрещены в любой части переносимого навыка,
   включая examples, fixtures и проверки.
9. Составной вопрос закрывается сочетанием возможностей, а не созданием
   отдельного навыка под формулировку вопроса.
10. Описание возможности должно позволять отличить ее от похожих возможностей
   до обращения к данным.

## 3. Диалоговые и общие возможности

| Capability ID | Продуктовая возможность | Основные входные значения | Ожидаемый результат |
| --- | --- | --- | --- |
| `CAP-CHAT-CONTEXT` | Продолжить вопрос с учетом предыдущего подтвержденного результата | Сессия, предыдущий результат, новое уточнение | Полный актуальный контекст без потери и случайной замены условий |
| `CAP-CHAT-CLARIFY` | Уточнить существенную неоднозначность | Вопрос, допустимые бизнес-трактовки | Один понятный уточняющий вопрос |
| `CAP-CHAT-NOT-FOUND` | Сообщить, что искомый объект или данные не найдены | Условия подтвержденной корректной выборки | Ответ об отсутствии данных с повторением существенных условий |
| `CAP-CHAT-OUT-OF-SCOPE` | Обработать вопрос вне области MVP | Пользовательский вопрос | Короткий отказ с описанием области ассистента |
| `CAP-CHAT-READ-ONLY` | Отклонить запрос на изменение данных | Команда пользователя | Понятное сообщение о режиме только для чтения |
| `CAP-CHAT-DEPENDENCY-ERROR` | Обработать недоступность DeepSeek или MCP | Вид зависимости, ошибка, идентификатор операции | Контролируемое сообщение без выдуманного бизнес-ответа |
| `CAP-COMMON-PERIOD` | Определить явный или относительный период | Дата, интервал, «вчера», «за месяц», «на текущий момент» | Нормализованные границы или момент времени |
| `CAP-COMMON-ENTITY` | Найти бизнес-объект по пользовательскому обозначению | Название, код, артикул, номер или штрихкод | Однозначный объект либо список кандидатов для уточнения |
| `CAP-COMMON-COUNT` | Подсчитать выбранные объекты | Набор объектов и условия | Количество с указанием того, что именно посчитано |
| `CAP-COMMON-AGGREGATE` | Рассчитать сумму, среднее, минимум или максимум | Показатель и набор строк | Значение с названием показателя и единицей |
| `CAP-COMMON-RANK` | Выбрать верхние или нижние позиции | Показатель, направление и лимит | Упорядоченный список с показателем ранжирования |
| `CAP-COMMON-DETAIL` | Получить реквизиты ранее найденного объекта | Ссылка на подтвержденный объект, перечень реквизитов | Запрошенные реквизиты именно этого объекта |
| `CAP-COMMON-GROUP` | Сгруппировать показатель | Показатель и разрез | Значения по периодам или бизнес-объектам |

## 4. Документация

| Capability ID | Продуктовая возможность | Основные входные значения | Ожидаемый результат |
| --- | --- | --- | --- |
| `CAP-DOC-SEARCH` | Найти материалы по теме | Формулировка вопроса и термины | Релевантные фрагменты доступных материалов |
| `CAP-DOC-TERM` | Объяснить термин УТ | Термин и контекст пользователя | Краткое определение с источником |
| `CAP-DOC-PROCEDURE` | Описать типовую операцию | Цель пользователя | Последовательность действий и существенные условия |
| `CAP-DOC-ERROR` | Объяснить ошибку или запрет операции | Текст ошибки и контекст | Возможные причины и проверяемые действия |
| `CAP-DOC-STATUS` | Объяснить состояние документа | Вид документа и статус | Бизнес-смысл статуса и допустимое следующее действие |
| `CAP-DOC-SOURCE` | Показать происхождение документационного ответа | Использованные материалы | Название, расположение, дата и тип источника |

Документационный корпус schema v1 состоит только из встроенной справки релиза
11.5.27.56. Внешние материалы не индексируются, не смешиваются с этим корпусом
и не используются для ответа.

## 14. Минимальное описание переносимой возможности

На продуктовом уровне переносимый навык обязан сообщать:

- стабильное назначение и версию;
- совместимую конфигурацию и релиз;
- бизнес-смысл входных значений;
- обязательность каждого входного значения;
- бизнес-смысл, поля и единицы результата;
- ограничения и условия корректного применения;
- зависимости от других возможностей;
- примеры подходящих и неподходящих вопросов;
- положительные и отрицательные контрольные сценарии.

Значения в переносимых examples и fixtures должны быть явно обозначены как
синтетические. Они служат только иллюстрацией или проверкой и не могут влиять на
выбор или исполнение навыка. Секреты и локальные пути запрещены в том числе в
этих разделах.

Точная структура JSON, способ загрузки, исполнения и связывания возможностей в
этом документе намеренно не определяются.
